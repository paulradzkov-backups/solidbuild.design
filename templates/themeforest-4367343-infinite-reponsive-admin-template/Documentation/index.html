Main structure

This template is built on twitter bootstrap 2.3.1 framefork.

If you are not familiar with bootstrap at all I suggest you first to visit this link and read it before any changes: Twitter bootstrap 2.3.1

This template includes 25+ HTML pages and over 30 jQuery plugins.

All css files are placed in folder named /css/. Inside folder /css/ you will find a directory plugin which contains css files for plugins.

All images are placed in folder named /img/. Inside folder /img/ you will find directories which accord to all plugins. For example, images for forms are placed in folder named /forms/ etc...

All jQuery plugins are placed in folder /js/. Inside folder /js/ you will find directories with plugins and in root folder /js/ you will find files like dashboard.js, main.js, forms.js etc.. These files contain all necessary code for plugins to work in demo pages. I don't suggest you using this files (create new one from scratch) as you will maybe not use all possibilities and plugins in this template so it will contain extra junk code.

All preview images are placed in folder /preview/.

</header>

<div class="mainNavigation">

</div>

<div class="content">

</div>

On this way we will create our header, our left sidebar - navigation and content.

Header

Header section is simple and you just need to replace it with your own elements. Demo example of header would look like this one:

<a href="#" class="logo"><img src="logoimage.png"></a>

<span id="mobileNav"><img src="img/mobile-icon.png"></span> /////// This will show a blue button on mobile devices (button which pulls navigation out and back)

<span id="phoneNav"><img src="img/mobile-icon.png"></span> /////// This will show an orange button on phone and the smallest devices (button which slidedowns phone's navigation from top and back)

<div class="phone-menu">
  <ul>
    <li><a href="#">This is phone navigation and display only on phone</a>
      <ul>
        <li><a href="#"><span>Submenu</span></a></li>
      </ul>
    </li>
  </ul>
</div>

<ul class="header-actions">
  <li><a href="#">icon image</a>
    <div class="dropdown">
    </div>
  <li>
</ul>

This will create header with logo, 2 buttons from which one displays on tablet devices (blue button) and another one which displays on phone (orange one). On tablet devices whole navigation and sidebare are pulled from side, while on phone it slidedowns another navigation from top.

Sidebar section

Sidebar section is simple to use. It contains main navigation with dropdown possibility on the left side and near that widgets area.  The structure looks like this one:

<div class="mainNavigation">
  <div class="innerNavigation">
    
    <div class="profile clearfix">
      <a href="#">user image</a>
      <div class="profile-options">
        <div class="info">
          <span class="info">Username</span>
          <span class="messages">8</span>
        </div>
        <div class="search">
          <div class="field">
            <input type="text">
            <a href="#" class="button-red">search icon image</a>
          </div>
        </div>
      </div>
    </div>

This will create user avatar with options and search field on the right side. Then we insert our main navigation:

<ul class="mainNav">
  <li class="active"><a href="#"><img src="image"> <span>Dashboard</span></li>
  <li class="dropdown"><a href="#"><img src="image"> <span>Dropdown</span>
    <ul>
      <li><a href="#"><span></span> Dropdown menu 1</a></li>
    </ul>
  </li>
</ul>

This is how the main navigation should look alike, it has also possibility for third child submenu, so just checkout demo pages for preview. Then last thing is sidebar:

    <div class="submenus">
      <div class="widget-holder">
        Place your widget here
      </div>
      <div class="divider"><span><i></i></span></div>
    </div>

  </div>
</div>

You can place all widgets inside widget-holder div, as on that way widget won't be touching edges of sidebar, because it will have 20px padding on sides. Div with class divider can be used to divide areas and split widgets from each other. For example widgets check out demo pages and you can copy/paste current code, move them, edit  etc... In sidebar you can use anything like forms, charts, carousel, pie charts, uploader widget, filetree etc...

Content area

Content area is simple for use. The structure is:

<div class="content">
  <div class="top-bar">
    
  </div>

  <div class="page-info">

  </div>

  <div class="inner-content">

  </div>
</div>

This is main structure for content area. So on the top we have topbar which can contain breadcrumbs on the left side and dropdown menus on the right side. Then we have page info which shows us which page is currently opened and also we can place for example quick actions on the right. Then we have the rest of the content page where widgets and boxes are placed (actually the rest of the content).

Top bar

  <div class="top-bar">
    <div class="breadcrumbs fLeft">
      <ul class="breadcrumb">
        <li><a href="#">Dashboard</a></li>
      </ul>
    </div>

    <ul class="bar-actions">
      <li class="dropdown"><a data-toggle="dropdown-toggle" href="#"> <img src="icon"> Some text</a>
        <ul class="dropdown-menu pull-right">
          <li><a href="#"><img src="icon"> Some submenu</a></li>
        </ul>
    </ul>
  </div>

By this way it will create us a breadcumbs on the left side and the top bar actions with dropdown possibility on the right side. Dropdown menu is used by bootstrap and it's easy to understand if you already visited twitter bootstrap page which we mentioned on the top of documentation.

Page info

<div class="page-info clearfix">
  <img src="current page icon">
  <h5>Page name</h5>


  <ul class="quick-actions">
    <li><a href="#"><img src="icon"></a></li>
  </ul>
</div>

This will create page name with an icon on the left side and quick actions on the right side. Quick actions work like switches, you can turn them on and off. It can be used for page auto refresh on/off and similar things...

Inner content

Inner content area can contain all elements like widgets, charts, forms, sparklines, tabs, etc...

Widget boxes

Widget boxes can contain anything including charts, text, tabs, tables etc.. Structure is following:

<div class="widget">
  <div class="widget-header">
    <h5>Some title in header<h5>
  </div>
  <div class="widget-content">
    This is widget content area
  </div>
</div>

Widget content area has 20px padding by default. You can remove that quick by adding class no-padding to the same. By that way padding will be 0px. We can also place dropdown buttons in header which be aligned on the right side, buttons without dropdown, tabs in content area, tabs in header area on the left-right side etc... Every demo page contains a different widget boxes styling so you can use them as examples and starting point.

Forms

For using forms please be ensure to include <form> element after using widget div. For example:

<div class="widget">
  <form class="form-horizontal">
    Here input widget header and content
  </form>
</div>

Forms are used on the following way:

<div class="form-row">
  <label class="control">Name of label</label>

  <div class="controls">
    <input type="text" class="span12">
  </div>
</div>

This will create one form row with label and input (full width). To change a grid of input just change class span12 to the grid you need. On this way you can also insert more inputs in the same line by using smaller grids. On the same way you insert selects, checkboxes, radios, multiselects, input tags, textarea, input file, spinners etc... If you would like to remove label, just place your element inside form-row div without any controls.

Form wizards

This template uses bootstrap wizards for form wizards.

<div id="root-wizard" class="wizard">
  <div class="steps">
    <ul>
      <li><a href="#tab1" data-toggle="tab"><img src="image"></a></li>
      <li><a href="#tab2" data-toggle="tab">1</a></li>
    </ul>
  </div>
  
  <div id="bar" class="progress progress-stripped"> <-- If you don't want to use remove this -->
    <div class="bar"></div>
  </div>

  <div class="tab-content">
    <div class="tab-pane" id="tab1">
      First step content
    </div>
    <div class="tab-pane" id="tab2">
      Second step content
    </div>
  </div>
 </div>

This is an example of form wizard with "icons and numbers as navigation". If you would like to place text instead of icons and numbers in navigation use following instead of div steps.

<div class="navbar">
  <div class="navbar-inner">
    <div class="container">
      <ul>
        <li><a href="#tab1" data-toggle="tab">First</a></li>
        <li><a href="#tab2" data-toggle="tab">Second</a></li>
      </ul>
    </div>
  </div>
</div>

This will create wizard with navbar navigation in header.

Gallery

Gallery is responsive using jquery masonry plugin.

You first create html structure:

<div class="gallery-container">
  <div clas="gallery-item">
    <img src="someimage.png">
  </div>
  ............
</div>

You repeat gallery-item as many times as you want to create more images. Then you simply call the plugin to gallery-container and viola! That's it, you have got a responsive gallery grid which aligns automatically as you change the size of window.
